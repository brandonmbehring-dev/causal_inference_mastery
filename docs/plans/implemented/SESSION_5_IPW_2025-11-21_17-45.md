## Target State

### New Module: `src/causal_inference/observational/ipw.py`

**Function**: `ipw_ate_observational(outcomes, treatment, covariates, **options)`

**Features**:
1. **Propensity Estimation**:
   - Logistic regression: `P(T=1|X)`
   - Return propensity scores and model diagnostics
   - Option to pass pre-computed propensities (for advanced users)

2. **Weight Trimming**:
   - Trim at percentiles (e.g., 1st/99th)
   - Option: `trim_at=(0.01, 0.99)`
   - Remove units with extreme weights

3. **Weight Stabilization**:
   - Stabilized weights: `SW = P(T) / P(T|X)`
   - Reduces variance, maintains unbiasedness
   - Option: `stabilize=True`

4. **Integration**:
   - Calls existing `ipw_ate()` with computed propensities
   - Reuses weight normalization and diagnostics

**Expected Lines**: ~200-300

### Phase 4: Layer 3 Monte Carlo Validation (1.5 hours)

#### DGPs with Confounding

**DGP 1: Linear Confounding**
```python
X ~ N(0, 1)
logit(P(T=1|X)) = alpha + beta*X  # Confounding
Y = tau*T + gamma*X + epsilon     # Outcome depends on X
```

**DGP 2: Quadratic Confounding**
```python
logit(P(T=1|X)) = alpha + beta*X + delta*X^2
Y = tau*T + gamma*X + epsilon
```

**DGP 3: Multiple Confounders**
```python
X1, X2, X3 ~ N(0, 1)
logit(P(T=1|X)) = alpha + beta1*X1 + beta2*X2 + beta3*X3
Y = tau*T + gamma1*X1 + gamma2*X2 + gamma3*X3 + epsilon
```

**Tests** (5000 runs each):
- Validate bias < 0.10 (observational, so slightly relaxed)
- Coverage 93-97%
- SE accuracy < 15%
- Compare trimmed vs untrimmed
- Compare stabilized vs non-stabilized

**Expected**: ~300 lines tests (3 DGP classes)

## Decisions Made

### Design Decisions

1. **Module Structure**: Separate `observational/` package
   - Rationale: Observational methods conceptually distinct from RCT
   - Allows clean separation of concerns
   - Future: Can add doubly robust, synthetic control, etc.

2. **Reuse RCT IPW**: Call `ipw_ate()` from RCT module
   - Rationale: Don't duplicate weight normalization, variance estimation
   - Single source of truth for IPW mechanics
   - Observational module focuses on propensity estimation

3. **Default: No Trimming, No Stabilization**
   - Rationale: Users should explicitly opt-in to modifications
   - Transparency: Clear what's being done
   - Research standard: Report both trimmed and untrimmed results

4. **Logistic Regression Only** (for now)
   - Rationale: Standard method, interpretable
   - Future: Can add machine learning methods (RF, XGBoost) later
   - Sufficient for Session 6 goals

5. **Monte Carlo Thresholds**: Relaxed vs RCT
   - Bias < 0.10 (vs 0.05 for RCT)
   - Rationale: Observational data has inherent bias from unmeasured confounding
   - IPW can only adjust for measured confounders

### Alternative Approaches Considered

**Alternative 1**: Modify `ipw_ate()` to accept covariates
- **Rejected**: Would make RCT function unnecessarily complex
- Better separation: RCT version stays simple, observational builds on it

**Alternative 2**: Use machine learning for propensity
- **Deferred**: Start with logistic regression (standard)
- Can add ML methods in future sessions

**Alternative 3**: Implement in RCT module
- **Rejected**: Observational methods conceptually distinct
- Cleaner to have separate `observational/` package

## Risks and Mitigation

### Risk 1: Propensity Model Misspecification
**Impact**: Biased ATE estimates
**Mitigation**:
- Provide diagnostics (AUC, pseudo-R²)
- Document limitations in docstrings
- Include specification tests in Monte Carlo

### Risk 2: Extreme Weights
**Impact**: High variance, unstable estimates
**Mitigation**:
- Weight diagnostics (max weight, CV)
- Trimming option
- Stabilization option
- Clear warnings in docstrings

### Risk 3: Collinearity in Covariates
**Impact**: Propensity model fails to converge
**Mitigation**:
- Error handling for singular matrix
- Diagnostics for collinearity
- Test with high-dimensional covariates
